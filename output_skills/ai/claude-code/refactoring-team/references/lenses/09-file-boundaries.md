# Lens: File Boundaries

Files are containers for related concepts. The boundaries between files should reflect the boundaries between ideas.

## The Question

Are the file boundaries in the right places? Do files contain coherent concepts, or arbitrary groupings?

## How to Spot

- Files named "utils," "helpers," "misc" — what concepts are hiding there?
- Concepts split across files that belong together
- Files doing multiple unrelated things
- Missing files — distinct concepts crammed into one file

## Process

Look at each file as a unit. What concept does it represent? Is that concept complete, or scattered? Are there concepts that deserve their own file?

## Go Deeper

What other boundary issues exist? What files should be split, merged, or renamed?
